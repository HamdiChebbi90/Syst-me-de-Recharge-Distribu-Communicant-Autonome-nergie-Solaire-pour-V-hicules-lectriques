Introduction

Contexte & Objectifs du Projet

La transition vers la mobilité électrique pose un défi majeur : les stations de recharge autonomes fonctionnant à l'énergie solaire sont limitées en puissance et en capacité de stockage. Sans un système de décision intelligent, les véhicules risquent d'être refusés, de subir des attentes excessives, ou de gaspiller l'énergie solaire disponible.

1

Concevoir un algorithme côté véhicule

Permettre à chaque EV de sélectionner la station optimale en fonction de son état réel (SOC, SOH), de sa position et des conditions de trafic.

2

Concevoir un algorithme côté station

Contrôle d'admission (ACA) avec déclassification de la recharge et prédiction de l'énergie solaire pour maximiser le taux d'acceptation.

3

Modéliser formellement le système

File d'attente M/M/C/N avec chaîne de naissance-mort, formules analytiques de P_refus, L_q et W validées par simulation.

4

Adresser sécurité et communication

Authentification PKI, chiffrement, anti-rejeu, gestion des pertes de paquets et architecture distribuée V2I/I2I/Cloud.

Partie 2 — Modélisation du Système

Modèle de File d'Attente — M/M/C/N

Notation de Kendall
M / M / C / N
MArrivées : Poisson (λ)
MService : Exponentielle (μ)
CServeurs : nombre de prises
NCapacité : prises + file
Pourquoi ce modèle ?
Processus stochastique
Les arrivées EV sont aléatoires — le processus de Poisson modélise ce comportement de manière exacte.
Propriété sans mémoire
La probabilité de finir le service ne dépend pas du temps déjà écoulé (distribution exponentielle).
Capacité finie N → refus mesurable
Quand le système atteint N, tout nouvel arrivant est rejeté. P_N est la métrique clé du taux de perte.
Facteur d'utilisation : $\rho = \dfrac{\lambda}{C \cdot \mu}$  |  Stabilité : $\rho < 1$

Probabilité de l'état vide P₀
$$P_0 = \left[ \sum_{n=0}^{C} \frac{(C\rho)^n}{n!} + \frac{(C\rho)^C}{C!} \sum_{n=C+1}^{N} \rho^{\,n-C} \right]^{-1}$$
Probabilité de refus P_N (taux de perte)
$$P_N = \frac{(C\rho)^C \cdot \rho^{\,N-C}}{C!} \cdot P_0$$
Nombre moyen en file L_q
$$L_q = P_0 \cdot \frac{(C\rho)^C \cdot \rho}{C!\,(1-\rho)^2} \left[ 1 - \rho^{\,N-C} \right]$$
Temps moyen d'attente W
$$W = \frac{L_q}{\lambda\,(1 - P_N)}$$
Partie 3 — Algorithmes de Décision

Algorithme Côté Véhicule — Sélection de Station

Pseudo-code
INPUT : SOC, SOH, GPS_pos, T_wait_max OUTPUT : Station* choisie // Capacité effective (SOH impacte la capacité) C_eff = C_nominal × (SOH / 100) e_cur = C_eff × (SOC / 100) autonomy = e_cur × η_km_per_kWh // Filtrer les stations atteignables stations ← getStationsInRange(GPS, autonomy) // Scorage multi-critères (coût à minimiser) for each s in stations: T_est ← s.waitTime + rechargeTime(SOC, SOH) if T_est > T_wait_max: skip score(s) ← w₁·d + w₂·T_est + w₃·Q(s) − w₄·E_avail(s) − w₅·free(s) if priority: score(s) −= bonus return argmin_s score(s)
Points clés vs approche basique
  SOH dans le calcul
La capacité effective = C_nom × (SOH/100). Un SOH=75% réduit l'autonomie réelle de 25% par rapport au SOC affiché.
  Vérification d'autonomie
Les stations hors de portée sont éliminées avant le scorage. Un véhicule à 15% avec SOH=70% ne peut pas viser une station à 10 km.
  Contrainte temporelle intégrée
Les stations dépassant T_wait_max sont exclues, pas pénalisées par un poids noyé dans la somme.
  Priorité ≠ simple poids
Bonus conséquent + traitement prioritaire en file (cf. ACA), pas un poids de 0.05 invisible dans le score total.

if Q(t) >= N_max: return Refuser C_eff = C_nom × (SOH/100) E_req = C_eff × (SOC_des − SOC_in) / 100 // Prédiction solaire pendant la recharge for rate in [Rapide, Normale, Lente]: T_svc = E_req / P_rate E_fut = estSolar(horloge, T_svc) + E_stored T_wait = estWait(Q(t), rate) if T_svc + T_wait ≤ T_wait_max and E_fut ≥ E_req: return (Accepter, rate) // Recharge partielle vers SOC_min E_part = C_eff × (SOC_min − SOC_in) / 100 if E_part ≤ E_stored: return (Partielle, Lente) return (Refuser/Rediriger)
Ce que l'approche basique rate
  Pas de déclassification
Accept/reject binaire. Un véhicule est refusé même si une recharge lente serait parfaitement faisable.
  Pas de prédiction solaire
L'énergie est une valeur statique. Notre ACA estime la production solaire pendant la durée de recharge.
  Pas de double contrainte
L'approche basique ne vérifie que l'énergie. Notre ACA vérifie temporelle ET énergétique simultanément.
  Seuil partiel arbitraire
Le « 30% d'énergie » n'est relié à aucun SOC_min. Notre seuil est calculé pour atteindre une autre station.

Algorithme d'assignation
INPUT : EV_pos, SOC_in, SOH, T_wait_max OUTPUT : Adresse station d_max ← autonomy(SOC_in, SOH) // Niveau 1 : V2I direct (~ms) cands ← queryLocal(EV_pos, d_max) // Niveau 2 : I2I voisin (~100ms) if cands.isEmpty(): cands ← queryNeighbors() // Niveau 3 : Cloud global (~1s) if cands.isEmpty(): cands ← queryCloud() // Sélection optimale best ← argmin(α·wait + β·distance) sendToEV(best.address) updateStatus(best)
V2I Communication directe
Le véhicule contacte les stations dans son rayon d'autonomie. Temps : ~ms.
I2I Coordination inter-stations
Si aucune station locale n'accepte, la station contacte ses voisines. Temps : ~100ms.
Cloud Fallback global
Dernier recours : requête au serveur central. Temps : ~1s.
  Tolérance aux pannes
Chaque niveau a un timeout. Échec V2I → I2I → Cloud → notification au véhicule.

Partie 5 — Simulation & Résultats

Analyse des Résultats

Impact du déclassification
Le processus de déclassification (Rapide→Lente→Partielle) réduit significativement P_refus, surtout lorsque λ approche la saturation. Sans déclassification, le taux de refus augmente brutalement dès ρ > 0.6.
Temps d'attente contrôlé
La contrainte T_total ≤ T_wait_max dans l'ACA garantit que W reste dans des limites acceptables même sous forte charge. Le file prioritaire réduit W pour les véhicules urgents.
Autoconsommation solaire
La prédiction E_solar(t) permet d'accepter des recharges que l'énergie stockée seule ne permettrait pas, augmentant le taux d'autoconsommation du photovoltaïque.
Impact des pertes de communication
À 5% de perte : impact négligeable. À 20% : P_refus augmente de 8-12 points car des véhicules ne reçoivent pas la réponse des stations à temps.
Overhead sécurité
L'authentification PKI ajoute ~50ms par requête. La détection de fraude (5% d'attaques) bloque les tentatives sans impact sur les véhicules légitimes.
Dimensionnement optimal
Pour λ=3, C=3, N=10 : P_refus < 5% avec déclassification. Pour λ=5, il faut C≥5 ou N≥15 pour maintenir P_refus < 10%.

Processus de Déclassification

Lorsqu'un véhicule demande une recharge que la station ne peut pas fournir (manque d'énergie ou dépassement du temps max), le système tente progressivement des modes de recharge inférieurs avant de refuser.

Rapide
P = 50+ kW
Normale
P = 7–22 kW
Lente
P ≤ 3.7 kW
Partielle
SOC → SOC_min
Refus / Redirection
Vers autre station
  Rapide → Normale
Si l'énergie solaire est insuffisante pour une charge rapide, réduire la puissance diminue le pic de consommation.
  Normale → Lente
Si même la normale dépasse le seuil temporel ou énergétique, la lente étale la charge sur une durée plus longue.
  Lente → Partielle
Dernier compromis : recharger juste assez pour atteindre une autre station au lieu de refuser.

Allocation Distribuée EV ↔ Stations

Algorithme d'assignation
INPUT : EV_pos, SOC_in, T_wait_max OUTPUT : Adresse de la station sélectionnée // 1. Le véhicule envoie sa position sendToServer(EV_pos, SOC_in, T_wait_max) // 2. Calculer l'autonomie maximale d_max ← autonomy(SOC_in, SOH) // 3. Interroger les stations à portée (V2I) candidates ← [] for each station in range(EV_pos, d_max): resp ← queryStation(station, SOC_in) if resp.admission == Accepter: candidates.add(resp) // 4. Si aucune station locale (I2I) if candidates.isEmpty(): candidates ← queryNeighborStations() // 5. Si toujours rien (Cloud) if candidates.isEmpty(): candidates ← queryCloud() // 6. Sélectionner la meilleure best ← argmin(waitTime + α·distance) sendToEV(best.address) updateStationStatus(best)
3 niveaux de communication
V2I Véhicule → Station locale
Communication directe (Wi-Fi, DSRC, 5G). Le véhicule contacte les stations dans son rayon d'autonomie. Temps de réponse : ~ms.
I2I Station → Stations voisines
Si aucune station locale n'accepte, la station contacte ses voisines. Coordination distribuée sans passer par le Cloud. Temps : ~100ms.
Cloud Internet → Réseau global
Dernier recours si aucune station locale ou voisine ne peut servir. Requête au serveur central pour trouver une station plus éloignée.
  Tolérance aux pannes
Chaque niveau a un timeout. Si V2I échoue → I2I. Si I2I échoue → Cloud. Si tout échoue → le véhicule est notifié avec les stations les plus proches même si refusées.

Sécurité & Communication

Menaces identifiées
Faux véhicules (identité falsifiée)
Un acteur malveillant se fait passer pour un EV prioritaire pour obtenir une recharge.
Injection de paquets / Messages falsifiés
Modifier les paramètres SOC, SOH ou priorité dans la requête V2I.
Rejeu d'attaque (Replay Attack)
Réémettre une requête valide d'un véhicule précédent pour occuper une prise.
Déni de service (DoS)
Inonder la station de requêtes pour saturer la file et bloquer les véhicules légitimes.
Élévation frauduleuse de priorité
Modifier le champ "priorité" pour passer devant les autres véhicules.
Contre-mesures proposées
Authentification PKI / Certificats V2G
Chaque véhicule possède un certificat numérique signé par une autorité de confiance. La station vérifie l'identité avant tout traitement.
Chiffrement des messages V2I
Tous les échanges sont chiffrés (TLS/AES) pour empêcher l'interception et la modification des paramètres.
Signature numérique des requêtes
Chaque requête est signée avec la clé privée du véhicule. La station vérifie la signature avec la clé publique (anti-rejeu via timestamp/nonce).
Timeout & Retransmission adaptative
Gestion des pertes de paquets : retransmission avec backoff exponentiel, détection de station déconnectée.
Consensus distribué entre stations
Mécanisme de vérification croisée I2I pour détecter les incohérences (ex: un véhicule signalé simultanément dans deux stations).

import numpy as np import simpy # Paramètres M/M/C/N LAMBDA, MU, C, N = 2.0, 3.0, 3, 10 P_SLOW, P_NORM, P_FAST = 3.7, 22, 50 def ev_arrival(env, station, stats): """Génère les arrivées Poisson""" while True: yield env.timeout(np.random.exponential(1/LAMBDA)) soc = np.random.uniform(0.10, 0.50) soh = np.random.uniform(0.70, 1.0) ev = {'soc': soc, 'soh': soh, 'priority': np.random.choice([0,1])} env.process(admission_control(env, station, ev, stats)) def admission_control(env, station, ev, stats): """ACA avec déclassement""" if station.count >= N: stats['refused'] += 1; return e_req = calc_energy(ev['soc'], 0.80) for rate in [P_FAST, P_NORM, P_SLOW]: t_svc = e_req / rate if t_svc <= T_WAIT_MAX: with station.request() as req: yield req stats['served'] += 1 yield env.timeout(t_svc) return stats['refused'] += 1 env = simpy.Environment() station = simpy.Resource(env, capacity=C) stats = {'served':0, 'refused':0} env.process(ev_arrival(env, station, stats)) env.run(until=10000)
